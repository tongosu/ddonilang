## 범위

- Q19 산출물이다.
- 입력 보고서: `BROKEN_CHECKS_AUDIT_V1.md`(Q3), `UI_RUNNER_DEPENDENCY_MAP_V1.md`(Q4), `PACK_FAIL_TRIAGE_V1.md`(Q8), `BROKEN_CHECKS_ROOT_CAUSE_V1.md`(Q15), `UI_RUNTIME_LOAD_CHECK_V1.md`(Q16).

## 교차 카운트 방식

- 후보 항목 키는 보고서가 직접 지목한 파일/체크/모듈/팩 경로로 잡았다.
- Q3와 Q15는 같은 체크 파일을 같은 항목으로 병합했다. Q15는 문서 참조 행 단위 보고서지만, Q19 표에서는 체크 파일 단위로 묶었다.
- Q4는 `분류=러너전용` 또는 `분류=고아`인 모듈/러너만 후보로 집계했다. `제품` 행은 제외했다.
- Q8은 FAIL/TIMEOUT pack 99개를 모두 검토 후보로 집계하되, 원 보고서의 삭제 심사 후보가 0개이므로 처분 후보는 `보류`로 두었다.
- Q16은 제품 91개/동적의심 54개가 모두 런타임 로드 성공이라 신규 삭제·통합 후보를 만들지 않았다. Q4 제품 후보를 상쇄하는 검증 근거로만 사용했다.
- `교차 지적 보고서 수`는 위 기준으로 같은 항목을 지적한 입력 보고서 수다.
