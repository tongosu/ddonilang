## 조사 결과 — 예상보다 훨씬 많이 이미 랜딩되어 있음

- **D18(Option)**: `T?`/`없음` 처리는 이미 완전히 MUST로 잠겨 있다(`SSOT_LANG:4649-4665` PINSPEC-OPT-01/OPTION-USE-01). **새로 할 것 없음.**
- **D18(Result)**: `결과:고름씨 = { #성공(값) #실패(이유) }` idiom이 이미 **`§5.10.4A`(MUST)**로 랜딩되어 있다(`SSOT_LANG:2695-2712`). 문서에 명시적으로 "**generic result family 전체 closure는 후속 단계로 둔다**"고 적혀 있다 — 이게 정확히 D18이 메우려는 자리다. **패치 대상.**
- **D32(닫힌 갈래 값 통합)**: `§27.1.1`(MUST, `SSOT_LANG:4668-4673`)이 이미 `T?`/`참거짓`/`고름씨`를 "유한(닫힌) 경우의 수"로 묶어 완전성 검사+AOT 점프테이블을 명시해뒀다. **알림 갈래/상태머신 상태만 추가하면 됨.**

## 결정 사항(사용자 확정, 2026-07-06)

`결과:고름씨` idiom을 제네릭 `맺음<T>`로 승격하되, 태그명은 DR-001(순우리말 정본 수렴) 원칙대로 **`#잘됨`/`#안됨`을 정본, `#성공`/`#실패`를 입력 별칭**으로 둔다(정본화기가 변환, AST/해시에는 항상 `#잘됨`/`#안됨`만 남음). 타입 이름도 `맺음`이 정본, `결과`는 입력 별칭(이미 이렇게 결정되어 있었음, 유지).

## 패치 1 — §5.10.4A `결과` idiom → `맺음<T>` 제네릭 승격

**파일:** `docs/ssot/ssot/SSOT_LANG_v24.12.9.md`

**L2695-2712 찾기(현재 전문):**
```
#### 5.10.4A `결과` 관용구 — v23.2.0 canonical idiom

```ddn
결과:고름씨 = {
  #성공(값:셈수)
  #실패(이유:글)
}.

변환결과 <- "123a"~을 숫자로.
변환결과에 따라 {
  #성공(값)이면 { 값 보여주기. }
  #실패(이유)이면 { 이유 보여주기. }
}.
```

- `고름씨`는 예외 흐름을 대신하는 **결정적 결과 패턴**에도 쓴다. (MUST)
- `시도/붙잡기` 류 예외 키워드는 도입하지 않는다. (MUST)
- `결과` idiom 은 먼저 stdlib/교재 current line 으로 올리고, generic result family 전체 closure 는 후속 단계로 둔다. (MUST)
```

**바꾸기:**
```
#### 5.10.4A `맺음<T>` 제네릭 Result 타입 — v24.13.0 MUST (D18 확정, 커널 v1)

`맺음<T>`는 실패 가능한 연산의 결과를 나타내는 **제네릭 닫힌 갈래 값**이다. 내부적으로 `고름씨` 메커니즘(§5.10) 위에 구현되며, 사용자에게 별도 이론어(Result/Sum type)를 노출하지 않는다.

```ddn
맺음<T> = {
  #잘됨(값:T)
  #안됨(사유:글)
}.

변환결과:맺음<셈수> <- "123a"~을 숫자로.
변환결과에 따라 {
  잘되면(값) { 값 보여주기. }
  안되면(사유) { 사유 보여주기. }
}.
```

- **정본 태그**: `#잘됨(값:T)` / `#안됨(사유:글)`. **입력 별칭**: `#성공` → `#잘됨`, `#실패` → `#안됨` (정본화기가 변환, AST/해시/전송은 항상 `#잘됨`/`#안됨`만 사용). *(DR-001 정합)*
- **정본 타입 이름**: `맺음<T>`. **입력 별칭**: `결과` → `맺음`(제네릭 파라미터 없이 쓰인 기존 `결과:고름씨 = {...}` 선언도 계속 입력으로 허용, 정본화기가 `맺음<T>`로 승격).
- **분기 설탕**: `잘되면(값) { ... }` / `안되면(사유) { ... }`는 `#잘됨(값)이면`/`#안됨(사유)이면`의 정본 축약형이다(§27.1.1 `~에 따라` 분기 규칙 그대로 적용).
- `맺음`는 예외 흐름을 대신하는 **결정적 결과 패턴**이다. (MUST)
- `시도/붙잡기` 류 예외 키워드는 도입하지 않는다. (MUST)
- **v24.13.0 갱신**: `결과` idiom의 generic closure를 `맺음<T>`로 확정한다(이전 판의 "후속 단계로 둔다"는 유예를 해제). 커널 v1 스펙(`PROPOSAL_LANG_KERNEL_V1_SPEC_20260704.md`)의 D18 확정 사항을 반영한다. (MUST)

회귀 pack(필수; 신규 또는 기존 확장):
- `결과`/`#성공`/`#실패` 입력이 `맺음`/`#잘됨`/`#안됨`으로 정본화되는지 확인하는 pack 1건 추가 권장(별도 Codex 브리프로 후속 진행).
```

## 패치 2 — §27.1.1 "유한(닫힌) 경우의 수" 목록에 맺음/알림/상태머신 추가

**파일:** `docs/ssot/ssot/SSOT_LANG_v24.12.9.md`

**L4672 찾기:**
```
- 대상: `T? (= T | 없음)`, `참거짓(참|거짓)`, `고름씨(#태그 …)` 등 **유한(닫힌) 경우의 수**를 가진 타입
```

**바꾸기:**
```
- 대상: `T? (= T | 없음)`, `참거짓(참|거짓)`, `고름씨(#태그 …)`, `맺음<T> (잘됨|안됨)`, 알림 갈래(`"KIND"라는 알림`), 상태머신 상태(설계 고정 항목, §V19 `세움{}`/`상태머신{}` 랜딩 시 적용) 등 **유한(닫힌) 경우의 수**를 가진 타입
```

> 참고: 상태머신 상태는 `세움{}`/`상태머신{}` 자체가 아직 구현 보류(DR-179/180/181, "설계 고정/구현 보류")이므로, 이 목록에 이름만 올려두고 실제 완전성 검사 집행은 그 랜딩 시점으로 미룬다. 이번 패치는 목록 등재만 한다.
