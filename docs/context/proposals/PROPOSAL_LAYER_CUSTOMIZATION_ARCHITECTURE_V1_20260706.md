# PROPOSAL_LAYER_CUSTOMIZATION_ARCHITECTURE_V1 (2026-07-06, 초안)

> 작성: Claude / 상태: 초안 — 사용자 확정 대기
> 관계: `PROPOSAL_ECOSYSTEM_LAYER_CONTRACT_V1_20260705.md`(D39~D41)의 후속. 그 문서가 "경계(계약)"를 다뤘다면 이 문서는 "경계 위의 커스텀 체계"를 다룬다.
> 실행 범위: 설계만. SSOT 반영은 사용자, 구현은 AGE 게이트에 따라 별도 착수. 문법 표기는 예시(sketch)이며 최종 표면 문법은 SSOT 개정 시 확정한다.

## 0. 문제 정의

사용자 질문 3개가 이 문서의 동기다:
1. "이야기만 DDN이면 나머지는 영원히 다른 언어로 짜야 하나?" → 아니다. 3단계 구분(결정론 저작/DDN 저작 표면/네이티브 접점)으로 답함.
2. "그러면 너무 좁은 시나리오 DSL 아닌가?" → 보개(`모양{}`)/슬기(`슬기.의도{}` 보류)는 이미 저작 표면이 있(었)다.
3. "샘/누리/거울은 `모양{}` 같은 커스텀 가능한 게 없잖아." → 맞다. 샘의 `매김{}`은 정적 메타(범위/간격/기본값/단위/이름표/조종 여부, `SSOT_LANG:7746`)일 뿐 생성형이 아니다. **이 빈 자리를 채우는 통일 체계가 이 문서다.**

## 1. 통일 원칙 — 모든 계층 커스텀은 같은 3층 패턴을 따른다 (D44)

| 층 | 이름 | 역할 | 언어 | 결정론 |
|---|---|---|---|---|
| A | **선언 블록** | "무엇을" — 계층에게 시킬 일을 선언 | DDN | 선언 자체는 정본 AST의 일부 |
| B | **씨 생성기** | "어떻게, 재사용 가능하게" — 매개변수를 받아 값을 만드는 재사용 로직 | DDN | **DDN으로 쓰는 순간 자동 결정론** |
| C | **네이티브 접점** | 원시 동작(픽셀 칠하기, LLM 호출, OS 이벤트, 소켓) | Rust/JS/WASM | 비결정론 허용 — 고정 계약으로만 통과 |

핵심 통찰: **DDN에는 ambient I/O가 없고 Fixed64 결정론 산술만 있으므로, 커스텀 로직을 DDN으로 쓰는 순간 자동으로 결정론적이다.** 따라서 "커스텀을 허용하면 결정론이 깨진다"는 걱정은 B층에서는 성립하지 않는다 — 비결정론은 오직 C층(네이티브)에만 존재할 수 있고, C층은 고정 계약(InputSnapshot/Patch/ObservationManifest/거울 기록 포맷)으로만 통과한다.

씨 가족은 이미 언어에 선례가 있다: `알림씨`(payload 스키마), `모양씨`(`(매개변수...) 이름:모양씨 = 모양{...}.`, `SSOT_TERMS:1279`), `겹보기씨`(overlay layer). 이 문서는 그 가족을 확장하는 것이지 새 패턴을 발명하는 게 아니다.

### 2.2 샘씨 — 입력 변환 생성기 (D45, 신설)

**동기:** 조이스틱 데드존 보정, 여러 입력원 가중 합성, 외부 API 응답→파라미터 매핑 같은 커스텀 입력 변환 로직을 지금은 DDN으로 쓸 수 없다. AI Gym(관찰 전처리)과 게임(컨트롤러 매핑)에서 반드시 필요해진다.

**의미론:**
- `샘씨`는 원시 입력값을 받아 정규화된 입력값을 돌려주는 **순수 DDN 함수**다. world 접근 불가(읽기도 불가 — 샘은 이야기 시작 전이므로).
- 실행 시점: 샘의 `begin_tick` 동결 **직전**. 변환 결과가 `InputSnapshot`에 실린다.
- **재연 규칙(핵심):** 거울은 **변환 후 값**(InputSnapshot에 실린 값)을 기록한다. 재연 시 샘씨를 재실행하지 않고 기록값을 그대로 쓴다. → 샘씨 버전이 바뀌어도 과거 기록의 재연 무결성이 유지된다. 기존 원칙 "재연 시 원천 재호출 금지"(`SSOT_MASTER:686`)와 정합.

**표면 문법 sketch (모양씨 선례 준용):**
```
(원시: 수) 조이스틱보정:샘씨 = {
  일때 크기(원시) < 0.1 { 되돌림 0. }
  되돌림 원시.
}.

채비 {
  추력: 수 = (0) 매김 { 범위: -1..1. 간격: 0.01. 원천: 조이스틱.축1 ~ 조이스틱보정. }.
}
```
`매김`에 `원천: <입력경로> ~ <샘씨>` 키 하나를 추가하는 것이 최소 침습 경로다(매김은 이미 "구조적으로 보존해야 하는 canonical structured meta", DR-173).

### 2.3 슬기씨 + 슬기.의도{} (D46, 확정 — "둘 다 가능하게" 반영)

**결정(2026-07-06):** 사용자 질문 "둘 다 가능하게 할 수 없어?"에 대한 답 — 그렇다, 둘 다 가능하다. 나눠서 볼 두 가지가 있었다: (a) 보류를 지금 풀지 여부, (b) 세부 설계를 지금 확정할지 여부. 이 둘은 서로 막지 않는다 — D45(샘씨)에 이미 적용한 "설계 고정 / 구현 보류" 패턴을 여기도 그대로 쓰면 된다:
- **지금 확정:** `슬기.의도{}` 보류 해제(폐기가 아니라 계속 논의 중이었다는 것을 공식화) + `슬기씨` 개념·표면 문법 방향을 설계로 못박는다.
- **나중으로 미루는 것:** 실제 파서/런타임 구현 착수 — D40/D41 수리(관찰자 API 분리, 6원천 enum)가 먼저 코드로 착륙해야 `슬기.의도{}`가 그 위에 자연스럽게 얹힌다.

즉 "설계 확정은 지금, 구현 착수는 D40/D41 이후"로 두 요구를 동시에 만족시킨다.

**구조:** 생성(LLM 호출)은 C층 네이티브 — 비결정론 허용. 그 앞뒤가 DDN이다:
- **`슬기.의도{}`(A층):** lesson이 슬기에게 "무엇을 물어볼 수 있는지"를 선언. 보류됐던 블록(`SSOT_LANG:129`)의 재검토. 선언이 없으면 슬기는 해당 lesson에 개입할 수 없다(기본 닫힘).
- **`슬기씨`(B층):** 슬기가 돌려준 제안을 **수용/거절/수정하는 결정론 필터**. 같은 제안이 오면 항상 같은 판정 — Gatekeeper의 lesson-측 확장이다.
- 수용된 제안만 `SeulgiPacket`으로 샘에 들어가 기록된다(기존 경로 유지, Q28/Q29 실측 확인). 재연은 기록된 packet만 사용 — LLM 재호출 없음.

**표면 문법 sketch:**
```
(제안: 슬기제안) 안전필터:슬기씨 = {
  일때 제안.행동갈래 == "보기만" { 되돌림 참. }
  되돌림 제안.대상 == "추력" 그리고 제안.값 <= 1.
}.
```

### 2.4 거울씨 + 설정.거울 확장 (D47)

**동기:** "무엇을 기록할지"의 콘텐츠는 이미 이야기가 커버하지만, **기록에서 파생되는 요약/진단**(예: "이번 실행에서 에너지 보존 오차 최대값", 교사용 리포트 항목)을 lesson 저작자가 정의할 방법이 없다.

**구조:**
- **`설정.거울`(A층, 기존 slot 확장):** 기록 정책(보관 tick 수, 진단 채널 on/off, 거울씨 등록).
- **`거울씨`(B층):** 거울 기록(읽기 전용)을 입력으로 요약값을 만드는 순수 DDN 함수. `state_hash` 밖(DR-152 정합), 사후 재계산 가능. D40 관찰자 계약의 "거울 소비 도구"가 DDN 안으로 들어온 형태다.

**표면 문법 sketch:**
```
(기록: 거울기록) 에너지오차요약:거울씨 = {
  되돌림 최대값(기록.줄마다("에너지오차")).
}.
```

### 2.5 누리 — 병합 알고리즘은 고정, 초기화/리셋 정책은 이미 존재 (D48, 정정)

**정정(2026-07-06):** 최초 초안은 "누리는 커스텀 표면이 아예 없다"고 썼으나 이는 틀렸다. 사용자 지적으로 확인:

- **`설정.누리`**(`SSOT_TERMS_v24.12.9.md:1273`): seed 등 누리 초기 메타. "결과 상태에는 간접 영향 가능"이라고 이미 명시되어 있다.
- **reset kind 5종**(`SSOT_LANG_v24.12.9.md:7975`, `SSOT_PLATFORM_v24.12.9.md:780`): `마당다시 / 판다시 / 누리다시 / 보개다시 / 모두다시` — 초기화 범위가 다른 이미 설계된 다중 옵션.
- `reset(seed, options)=모두다시`(NuriGym, `SSOT_PLATFORM_v24.12.9.md:586`) — 이미 seed+options로 파라미터화됨.

**구분해야 할 두 가지:**
1. **patch 병합 알고리즘 자체**(어떤 patch를, 어떤 순서/규칙으로 world에 반영하는가) — 이건 여전히 고정이어야 한다. 이걸 lesson마다 바꾸면 같은 코드가 엔진에 따라 다른 결과를 내게 되어 결정론 계약이 붕괴한다. **여기는 D48의 "커스텀 없음" 원칙이 그대로 유지된다.**
2. **초기화/리셋 정책**(seed, reset 범위) — 이건 이미 존재하는 A층 저작 표면이다. `설정.누리`가 그 자리이고, reset kind 5종이 그 옵션 목록이다. B층(생성기, "누리씨")은 필요 없다 — reset 정책은 "값을 선택하는 선언"이지 "매 tick 재사용할 생성 로직"이 아니기 때문이다(모양씨/샘씨처럼 매개변수마다 다른 결과를 계산하는 함수가 아니라, lesson 시작 시 한 번 고르는 값이다).

**수정된 결론:** 누리는 5계층 중 "B층(씨)이 없는" 유일한 계층은 맞지만(병합 알고리즘은 진짜 고정), "A층(선언)이 없는" 계층은 아니다 — `설정.누리`+reset kind 5종이 이미 A층을 채우고 있다. D44 표는 이를 반영해 정정한다.

### 2.6 보개 — 기존 로드맵 유지

`모양{}`(primitive-only, view-only)/`모양씨`/`겹보기씨`가 이미 설계 고정, AGE4 진행 중. 이 문서는 새 설계를 추가하지 않고, C층 접점을 D40 관찰자 계약(ObservationManifest 선언 대상만 읽기, 누리 쓰기 금지, state_hash 무영향)으로 못박는 것만 연결한다.
