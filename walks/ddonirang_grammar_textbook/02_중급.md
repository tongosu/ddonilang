# 02 중급 — 중급 계층

exposure_level: `중급`  
전제 지식: [01_기초.md](01_기초.md) 완료  
완결 기준: 흐름씨와 조건 훅을 함께 쓰는 시뮬레이션을 작성할 수 있다.

---

## 2-1. 조건 분기 — 만약/이라면

```ddn
(매마디)마다 {
  만약 속도 > 0 이라면 {
    위치 <- 위치 + 속도.
  }.
  아니면 {
    위치 <- 위치.
  }.
}.
```

---

## 2-2. 반복 — ~에 대해

```ddn
(매마디)마다 {
  (i) 0..5에 대해 {
    합계 <- 합계 + i.
  }.
}.
```

---

## 2-3. 매김 {} — 슬라이더 힌트

```ddn
채비 {
  g: 수 = (9.8) 매김 { 범위: 1..20. 간격: 0.1. }.
  dt: 수 = (0.05) 매김 { 범위: 0.01..0.1. 간격: 0.01. }.
}.
```

- `매김 {}` 이 있는 항목만 UI 슬라이더 생성
- `범위: a..b.` : 슬라이더 최솟값..최댓값
- `간격: s.` : 슬라이더 스텝

---

## 2-4. 흐름씨 (<<-) — 고정점 수렴 대입

흐름씨는 **파생 값**을 선언한다. 다른 변수가 바뀌면 자동으로 재계산된다.

```ddn
채비 {
  입력: 수 <- 0.
  두배: 수 <- 0.
}.

(시작)할때 {
  입력 <- 5.
}.

(매마디)마다 {
  두배 <<- 입력 * 2.
}.
```

### 흐름씨 실행 위상 (중요)

한 마디(tick) 안의 실행 순서:
```
[1단계] ordinary 대입 (<-)
[2단계] 흐름씨 고정점 수렴 (<<-)
[3단계] 조건 훅 실행 (됩때 / 인 동안)
```

이 순서는 변경 불가다.

---

## 2-5. 됩때 — (조건)이 될때 {}

`(조건)이 될때 {}`는 조건이 **거짓→참으로 전환**되는 순간 1회 실행된다.

```ddn
채비 {
  카운터: 수 <- 0.
  값: 수 <- 0.
}.

(시작)할때 {
  값 <- 3.
}.

(매마디)마다 {
  값 <- 값 + 1.
}.

(값 > 10)이 될때 {
  카운터 <- 카운터 + 1.
}.
```

> **됩때는 중급이다.** 이유:
> - 흐름씨 [2단계] 완료 후 실행됨을 이해해야 한다 (위상 규칙).
> - 훅 본문에서 흐름씨 변수를 바꿔도 같은 마디의 흐름씨가 다시 발화되지 않는다 (P2 규칙).

---

## 2-6. 인 동안 — (조건)인 동안 {}

`(조건)인 동안 {}`은 조건이 **참인 모든 마디**마다 실행된다.

```ddn
(카운터 < 5)인 동안 {
  값 <- 값 + 1.
}.
```

| 훅 | 실행 조건 |
|---|---|
| `(조건)이 될때 {}` | 거짓→참 전환 순간 1회 |
| `(조건)인 동안 {}` | 참인 매 마디마다 |

---

## 2-7. 중급 완전 예제 — 임계값 감지

```ddn
#기본관찰: 카운터

채비 {
  t: 수 <- 0.
  입력: 수 <- 0.
  흐름_값: 수 <- 0.
  카운터: 수 <- 0.
}.

(시작)할때 {
  입력 <- 5.
}.

(매마디)마다 {
  흐름_값 <<- 입력 * 2.
  t <- t + 1.
}.

(흐름_값 > 8)이 될때 {
  카운터 <- 카운터 + 1.
}.

보임 {
  x축: t.
  값: 카운터.
}.
```

실행 흐름:
1. `입력 = 5` (시작할때)
2. 매 마디: `흐름_값 = 10` (<<- 고정점)
3. 됩때 조건 `흐름_값 > 8` → 참 → `카운터 += 1`
