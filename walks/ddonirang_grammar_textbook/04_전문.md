
# 04 전문 — 전문 계층

exposure_level: `전문`  
전제 지식: [03_고급.md](03_고급.md) 완료  
완결 기준: proof / capability / 슬기 / 배움틀을 **설명층과 실행 경계**로 구분해 읽을 수 있다.

---

## 4-1. 이 문서의 역할

이 문서는 또니랑의 전문 계층을 **입문 surface 소개서**가 아니라,
아래 4축을 어디까지 current line으로 읽고 어디부터 follow-on / docs-first로 읽어야 하는지 정리하는 안내서다.

1. `밝히기 {}` / `증명 {}` 류의 proof 설명축
2. capability gate (`det_tier` / `openness` / `runtime_status`) 운영축
3. `슬기`의 의도 경계와 Gatekeeper 검문축
4. `배움틀`의 model artifact / infer-only / training follow-on 축

---

## 4-2. AGE 와 전문 노출은 같은 뜻이 아니다

전문 계층이라고 해서 항상 AGE가 늦다는 뜻은 아니고,
AGE가 빠르다고 해서 초급 노출이라는 뜻도 아니다.

예:

- `(조건)이 될때 {}` / `(조건)인 동안 {}` 는 runtime evidence 가 빠르지만 교육 노출은 중급이다.
- `슬기` 관련 문서는 전문 계층 설명축에 놓일 수 있지만, 실제 runtime/current line closure 는 별도 evidence 로 판정한다.

---

## 4-3. capability gate — 전문 계층의 읽는 법

전문 계층에서는 capability gate를 세 장부로 나눠 읽는다.

| 축 | 역할 |
|---|---|
| `det_tier` | 결정성 tier 계약 |
| `openness` | 프로젝트/가지가 요구하는 바깥 열림 수준 |
| `runtime_status` | 문법/기능이 실제 runtime current line인지, designed/docs-first인지 |

핵심은 이 셋을 섞지 않는 것이다.

- `D-ULTRA` 라고 해서 문법이 자동 승격되지 않는다.
- `open=record|replay` 라고 해서 world write 가 허용되는 것도 아니다.
- `runtime_status` 는 문법 closure ledger 이고, project policy 와 같은 표가 아니다.

---

## 4-4. 슬기 — 모델 이름이 아니라 경계

전문 계층에서 `슬기`는 특정 모델 이름이 아니라,
**외부 참모/의도 계층이 세계에 들어오기 전의 정규화 경계**로 읽는다.

핵심 current line:

- `슬기`는 곧바로 세계를 바꾸지 않는다.
- world-affecting 변화는 계속 `InputSnapshot` 경계를 탄다.
- Gatekeeper 는 reject / pass / trace 정책을 분리한다.
- raw prompt/response, debug trace 는 `state_hash` 밖에 둔다.

즉, 전문 계층에서 중요한 것은 “무슨 모델을 쓰는가”보다
**어디에서 봉인되고 어디에서 검문되는가** 다.

---

## 4-5. 배움틀 — 공방 층

`배움틀`은 코어 runtime 안의 새 마법문이 아니라,
모델 제작·학습·봉인·배포를 담당하는 **공방/toolchain 층**으로 읽는다.

현재선에서 먼저 보는 것은 full training DSL이 아니라 아래 minimum 이다.

- `std_model_artifact` minimum
- `model_artifact_inference_v1` infer-only deterministic path
- artifact seal/hash/det-tier mismatch fail

즉, 전문 계층의 첫 목표는 “모델을 마구 학습시키는 문법”이 아니라,
**같은 artifact + 같은 input = 같은 output** 을 지키는 바닥을 이해하는 것이다.

---

## 4-6. 이 문서가 주장하지 않는 것

이 문서는 다음을 주장하지 않는다.

- 새 core keyword 승격
- 슬기/배움틀 full runtime closure 완료
- 자연어 슬기 surface current-line 승격
- full AI training DSL landed
- proof/capability 관련 product completion

이 문서는 **전문 계층 설명 shell** 이다.
현재선의 actual closure 판정은 계속 pack/checker/CI PASS evidence 로만 한다.
