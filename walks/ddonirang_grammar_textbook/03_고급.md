# 03 고급 — 고급 계층

exposure_level: `고급`  
전제 지식: [02_중급.md](02_중급.md) 완료  
완결 기준: 임자-알림씨 메시지 흐름과 계약 물림을 이해하고 작성할 수 있다.

---

## 3-1. 임자 {} — 상태 주체 정의

임자는 고유한 상태를 가진 주체다.

```ddn
로봇:임자 = {
  체력: 수 <- 100.
  위치: 수 <- 0.
}.
```

- `임자` = 주체 정의
- `제` = 임자 본문 안에서만 유효한 자기참조

---

## 3-2. 알림씨 (~~>) — 임자 간 메시지

```ddn
(공격) 공격신호 ~~> 상대방.
```

- `~~>` : 알림 송신
- 수신: `"공격신호"라는 알림이 오면 {}`

```ddn
로봇:임자 = {
  체력: 수 <- 100.

  "공격"라는 알림이 오면 {
    제.체력 <- 제.체력 - 10.
  }.
}.
```

---

## 3-3. 상태머신 {} / 바뀔때마다

```ddn
신호등:임자 = {
  상태머신 {
    처음: 빨강.
    빨강 -> 초록.
    초록 -> 노랑.
    노랑 -> 빨강.
  }.

  (지금상태)바뀔때마다 {
    // 상태 전이 시 실행
  }.
}.
```

> `전이마다`는 `바뀔때마다`의 입력 별칭이다.

---

## 3-4. 계약 {} / 물림

계약 블록은 전제/후행 조건을 검사한다.

```ddn
계약 {
  전제: 체력 > 0.
  후행: 체력 <= 최대체력.
  중단 이라면 {
    // 계약 위반 시 물림(롤백) 후 실행
    오류 <- 1.
  }.
}.
```

- `중단` : 조건 위반 시 계약 구역 진입 시점 상태로 물림(Snapshot/Restore)
- `알림` 모드: 물리지 않고 진단만

---

## 3-5. 덩이 {} — 원자 커밋 블록

```ddn
덩이 {
  A <- A + 1.
  B <- B - 1.
}.
```

> `덩이 {}`는 AGE0에 구현됐으나 **exposure_level = 고급**이다.  
> 계약/거래 맥락에서만 필요하기 때문이다.

---

## 고급 계층 구문 목록

| 구문 | exposure_level | AGE |
|---|---|---|
| `이름:임자 = {}` | 고급 | AGE0 |
| `"KIND"라는 알림이 오면 {}` | 고급 | AGE0 |
| `(payload) 알림씨 ~~> 받는이.` | 고급 | AGE1 |
| `상태머신 {}` / `바뀔때마다` | 고급 | AGE2 |
| `계약 {}` / `물림` | 고급 | AGE0 |
| `덩이 {}` | 고급 | AGE0 |
| 후치 표지 (`}한것`, `}인것`, `}아닌것`, `}하고`, `}해서`) | 고급 | AGE0 |
| `미루기` | 고급 | AGE1 |

> AGE가 빨리 구현됐어도 노출 계층은 고급이다 (AGE ≠ exposure_level 원칙).
