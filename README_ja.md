# 또니랑 (Ddonilang)

> ゲームとAIの時代のための **韓国語ネイティブ** プログラミング言語・ツール  
> **決定性（再現性）** は「オプション」ではなく **文法** に組み込む

![또니랑 로고](publish/assets/ddonirang_wordmark.png)

---

## プロジェクト概要

또니랑は「考えがそのままゲームになる言語」を目指す **韓国語中心** の言語・ツールプロジェクトです。

- **韓国語ネイティブ**: 韓国語の語順/表現をそのままコードにし、曖昧さは正本化（キャノン）で減らします。
- **決定性（Determinism）**: 同じ入力ならどこでも同じ結果になるよう設計。  
  ゲームの不具合再現、実験比較、強化学習の信頼性のために「再現」をデフォルトにします。
- **AI志向 + AI‑aware**: AIが書いたコードでも「信頼できる」ように  
  **팩(pack)テスト + 正本化ゴールデン比較 + リプレイ/トレース** を組み合わせます。
- **教育/創作フレンドリー**: ゲームだけでなく、数学・物理・経済を「実験」に変える道具へ拡張します。
- **呪文世界（Ddonilang世界）**: 言葉（呪文）で世界を変えるデモ/チュートリアル世界観を整備中です。

今は **姉と一緒に「셈그림」（動く数学の絵）** を作りながら、  
「学びになる創作」がどうあるべきかを検証しています。

## 決定性はエンジンのオプションではなく文法

또니랑は **曖昧な解釈の余地** を文法で先に減らします。  
そのため「人が書いてもAIが書いても」意図が同じ形に収束します。

主な設計選択:

- **助詞境界 `~`（正本表記）**: 助詞の境界をコードで明示。  
  例: `扉~を`, `呪文~で`, `私~は`
- **単一代入 `<-`**: 状態更新の表記を一つに統一。
- **定義と実行の分離**: 定義は `:動き = { ... }`、実行は **動詞の基本形**（例: `表示する`, `開ける`）。
- **単一連結 `して`**: 複数動作の接続を一つに収束。
- **契約/ガード**: 違反は「無視」ではなく診断/記録（鏡）に残す。

```ddn
基盤.扉_状態 <- "閉".

  { 基盤.扉_状態 == "閉" }もの なら {
    基盤.扉_状態 <- "開".
    呪文を表示する.
  } それ以外 {
    "もう開いています。"を表示する.
  }.
}.

"開け！"で 扉を_開ける.
```

## 또니랑の6要素契約

또니랑は「世界」を6つの層で捉えます。

| 名前 | 役割 | 一言 |
|---|---|---|
| 源（샘） | 入力スナップショット | すべての行動は入力から始まる |
| 世界（누리） | 世界状態 | ゲーム/シミュレーションの状態 |
| 物語（이야기） | 規則・進行 | 何がなぜ変わるかを説明する層 |
| 可視界（보개） | 画面・音 | 見える界 — 表現(2D/3D/サウンド) |
| 鏡（거울） | 記録・リプレイ | 巻き戻し/監査/再現 |
| 知恵（슬기） | AI(補助/守り) | AIが助け、ときに守る |

## AIとの協業方式

또니랑は「AIがコードを書く」ことを前提に設計します。  
ただしコアは非決定化せず、**検証可能な流れ**を作ります。

- 人（私）：意図/説明/基準を決める
- AI：コード生成 + アイデア提案 + 改善提案
- **팩(pack)テスト**：例が正本（ゴールデン）を通り続けるか確認
- **正本化（キャノン）**：表記の揺れを1つの形に収束
- **リプレイ/トレース**：問題をそのまま再現/復元

## aiGYM — 決定的AI訓練/実験環境

aiGYMは 또니랑の **決定性ベースの訓練/実験環境** です。  
同じ入力・シードなら **必ず同じ結果** を得られ、学習/比較/デバッグが容易になります。

- 入力/ログ/リプレイを束ねて **再現可能な学習実験** にする
- 方針変更前後を **팩(golden)で比較**
- ゲーム・数学・物理・経済実験を **標準訓練インターフェイス** で接続

## 呪文世界（Ddonilang世界） — 呪文で開く世界

呪文世界（Ddonilang世界）は「言葉（呪文）で世界を変える」デモ/チュートリアル世界観です。

- 呪文（言葉）を **自然言語に近いコード** で書く
- 世界（누리）の状態が **決定的に変化** する流れを示す
- 初心者が「なぜ動くのか」を **リプレイ/トレースで確認**

## 学習に役立つ実験室（計画）

또니랑はゲームだけでなく、**学習を「実験」に変える道具**も作ります。  
この部分は「誰かが」一緒に作ってくれるかもしれません。

### 1) 셈그림（動く数学の絵）
- 数式/図形/物理法則を **「マディ・タイムライン」** で動かす
- 同じコード → 同じフレーム（再現）
- 数学を **見える化** する道具

### 2) 物理スタジオ（Physics 2D）
- 理論（式）↔ 実験（누리）↔ グラフ（比較）
- 自由落下/放物線/衝突などを「手で触る」実験

### 3) 経済学実験室（Eco‑Lab）
- レバー（スライダー）で政策/市場実験
- 市場/流れ/関係ビュー + 巻き戻し（復習）
